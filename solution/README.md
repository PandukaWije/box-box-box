# Box Box Box — Solution

## Strategy: Deterministic Formula Reverse-Engineering

This challenge looks like an ML problem but is actually a **parameter estimation problem**.
The simulation is deterministic — the same formula generates every race result.
The goal is to find the exact formula parameters, then simulate perfectly.

---

## Lap Time Formula

```
lap_time = base_lap_time
         + compound_offset[c]                            ← SOFT/MEDIUM are faster than HARD
         + deg_rate[c]                                   ← compound-specific degradation
           * max(0, tire_age - cliff[c])                 ← grace period before deg kicks in
           * (1 + temp_scale * (track_temp - 30))        ← higher temp = more degradation
```

**9 parameters to learn:**
| Parameter | Meaning |
|-----------|---------|
| `soft_offset` | SOFT is X seconds/lap faster than HARD (should be < 0) |
| `med_offset` | MEDIUM is X seconds/lap faster than HARD (should be < 0) |
| `soft_deg` | SOFT degradation rate (seconds lost per lap after cliff) |
| `med_deg` | MEDIUM degradation rate |
| `hard_deg` | HARD degradation rate |
| `soft_cliff` | Laps before SOFT starts degrading |
| `med_cliff` | Laps before MEDIUM starts degrading |
| `hard_cliff` | Laps before HARD starts degrading |
| `temp_scale` | How much temperature amplifies degradation |

## Step-by-Step Workflow

### 1. Explore the data (understand structure)
```bash
python solution/explore_data.py
```
This shows sample races, compound patterns, stint length distributions.

### 2. Fit parameters from historical data
```bash
python solution/fit_params.py
# Optional flags:
python solution/fit_params.py --fit-races 1000 --val-races 300
```
- Loads N historical races
- Runs global optimisation (Differential Evolution) + local refinement (L-BFGS-B)
- Saves `solution/fitted_params.json`
- Takes ~5-15 minutes depending on race count

### 3. Validate accuracy
```bash
python solution/validate.py --n 1000 --debug
```
- Tests against historical races (not test cases)
- Shows accuracy %, first 5 failures in detail

### 4. Run against test cases
```bash
./test_runner.sh
# Or manually:
cat data/test_cases/inputs/test_001.json | python solution/race_simulator.py
```

## Tuning for Higher Accuracy

If accuracy is below 90%, try:

1. **More fit races**: `python solution/fit_params.py --fit-races 2000`

2. **Different formula structure**: Edit `fit_params.py` → `lap_time_formula()`.
   Try quadratic degradation: `deg_rate * eff_age ** 1.5` or `deg_rate * eff_age ** 2`

3. **Per-compound temperature interaction**: Maybe temp affects compound offsets too:
   ```python
   offsets['SOFT'] += temp_on_soft * (track_temp - 30)
   ```

4. **Check the explore output**: If SOFT no-pitters consistently finish poorly even in short races,
   the compound offset might need adjustment.

## Key Insight

Positions are never directly predicted. The model:
1. Computes a **total time** per driver (regression)
2. **Sorts** drivers by time — lowest = 1st place

This guarantees no position ties and physically consistent results.
